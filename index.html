Xpoz Enrichment

Как работает enrichment в Xpoz и какие данные можно собирать

Эта страница описывает текущий пайплайн в xpoz/deploy: от входного списка аккаунтов до анализа, enrichment и сохранения результата в analysis_results.

Источник: accounts Результат: analysis_results CLI: batch_analyze.py One-off: analyze_account.py

1. Общая схема работы

Шаг 1. Берём входные аккаунты

Скрипт читает usernames из таблицы accounts. Для batch-режима можно использовать уже готовые выборки, например email-qualified: аккаунты с email и достаточным числом подписчиков.

Шаг 2. Обогащаем профиль через Xpoz

Для каждого username вызывается enrichment: профиль Instagram, посты, внешние ссылки и, при необходимости, best-effort данные по Twitter/X.

Шаг 3. Считаем критерии

Python считает performance и engagement-метрики, а Claude Haiku определяет сигналы монетизации по bio и captions.

Шаг 4. Сохраняем результат

Обогащённые данные и итоговые флаги записываются в analysis_results, после чего их можно фильтровать, экспортировать и использовать для поиска следующих кандидатов.

2. Что именно собирается из Xpoz

Блок Какие данные Зачем
Instagram profile id, username, fullName, biography, followerCount, followingCount, mediaCount, isVerified, isPrivate, externalUrl Понимание размера аккаунта, bio, статуса и внешней ссылки.
Instagram posts postType, mediaType, caption, createdAtDate, likeCount, commentCount, reshareCount, videoPlayCount Расчёт performance, low-performer stability и engagement rate.
Other socials Bio links, YouTube, Twitter/X и другие внешние соцсети, найденные в bio. Понимание мультиканальности и возможности дальнейшего enrichment.
Twitter/X best-effort username, name, description, followersCount, tweetCount, isVerified Дополнительный сигнал масштаба и внешней аудитории.

3. Какие критерии считаются сейчас

Критерий Что проверяется Где считается
C1 Reels performance Не менее 5 reels с views ≥ 150% от подписчиков. Python
C2 Low-performing reels Среднее нижних 10 reels из последних 15 > 15% от подписчиков. Python
C3 Post engagement Средний engagement по последним постам ≥ 1.5%. Python
C4 Monetization Есть ли сигналы монетизации: курсы, лид-магниты, комьюнити, вебинары, консультации. Claude Haiku
C5 Other socials Какие ещё соцсети и внешние ссылки есть в профиле. Xpoz + bio parsing

4. Что попадает в analysis_results

Ниже ключевые поля, которые уже сохраняются и пригодны для фильтрации.

username, analyzed_at, follower_count, posts_analyzed, reels_performance, reels_90d_count, reels_above_150pct, low_performing_reels, bottom10_avg_views, post_engagement, engagement_rate_pct, total_interactions, monetization, monetization_signals, monetization_reason, youtube_url, twitter_url, twitter_followers, other_socials, error, llm_cost_usd, xpoz_results_used

Это уже достаточно, чтобы строить shortlist качественных аккаунтов, сегментировать базу и запускать дополнительный enrichment только по перспективным профилям.

5. Какие данные можно собирать для поиска новых аккаунтов

Enrichment Xpoz полезен не только для оценки уже известных аккаунтов, но и для расширения базы. На практике можно собирать следующие типы данных:

  • Базовые profile attributes: размер аккаунта, verified/private status, media count, bio, external URL.
  • Контентные сигналы: caption-темы, тип контента, частота reels, средняя вовлечённость, стабильность просмотров.
  • Монетизация: наличие продуктов, курсов, консультаций, бесплатных воронок, communities, webinar CTA.
  • Cross-platform links: YouTube, Twitter/X, TikTok, Telegram, личные сайты, формы, landing pages.
  • Поисковые seed-поля: username, full name, biography keywords, niche words, external domain, схожие CTA.
  • Ретрай и enrichment quality: ошибки, quota-failures, пустые follower counts, real-time fallback results.

6. Как использовать enrichment для сбора новых аккаунтов

Подход A. Расширение от уже найденных аккаунтов

Берём уже хорошие аккаунты из analysis_results, смотрим bio, external domains, YouTube/Twitter links и повторяющиеся монетизационные паттерны. Это даёт новые поисковые seed'ы для ручного или автоматического поиска похожих аккаунтов.

Подход B. Поиск по username / niche words

Можно использовать username, full name и biography keywords как основу для поиска похожих профилей: creator, coach, agency, consultant, course, community, webinar, newsletter и так далее.

Подход C. Фильтрация по quality thresholds

Новый аккаунт имеет смысл добавлять в pipeline, если enrichment показывает достаточный follower count, не private-профиль, признаки reels-активности и хотя бы базовый monetization fit.

7. Практические примеры фильтров для новых кандидатов

  • High-fit creators: reels_performance = true, low_performing_reels = true, post_engagement = true
  • Monetized accounts: monetization = true
  • Multi-channel creators: есть youtube_url или twitter_url
  • Retry candidates: error is not null, но профиль перспективный по остальным полям
  • Expansion seeds: одинаковые домены, CTA, bio keywords и type-of-offer у уже успешных аккаунтов

8. Текущие ограничения

  • Некоторые поля могут приходить неполными, поэтому в коде уже есть real-time fallback по Instagram ID.
  • Twitter/X enrichment best-effort и не должен считаться обязательным.
  • Монетизация зависит от наличия ANTHROPIC_API_KEY; без него критерий C4 будет пропущен.
  • Текущий pipeline в основном анализирует уже известные usernames из accounts; для полноценного discovery новых аккаунтов нужен отдельный ingestion/search слой.

9. Что можно добавить следующим шагом

  • Отдельную таблицу discovery-кандидатов до попадания в accounts.
  • Сохранение seed-атрибутов: niche, bio keywords, domain, source search term, source account.
  • Автоматические enrichment-правила: если найден YouTube/Twitter/domain, запускать дополнительный lookup.
  • Сегментацию новых аккаунтов по offer type, audience size, language, geography и social mix.
